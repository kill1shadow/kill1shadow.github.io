父组件传props到子组件

有时候需要在一个第三方组件的基础上，套一层封装，挪过来作为自己内部的组件

封装涉及的逻辑，一般比较轻量，因为第三方组件功能都很健全，只需作点小改动，就能适应业务逻辑

第三方组件基础功能多，同时暴露的接口( props )也多，所以新组件需要原封不动把接口暴露出去

React、Tsx里的做法

  • jsx 语法支持解构赋值，把 props 从父组件全部传给子组件
  • ts 的 interface 语法，能把子组件的接口类型暴露出去

Vue中如何实现类似的 props 全部转发

重点在于以下:

  • 不使用 template 语法，改用更原始的 render()
  • render() 的第二个参数 context，相当于 option 里的 this
  • createElement 的第二个参数相当于子组件的 option
  • 从 context 里获取 props 对象，传递下去
  • 要开启 函数式组件 模式，把 functional 设为 true，否则 context 不会有值
  • attrs 是 HTML attribute，也需要转发下去
  • 组件实例化阶段，attrs 会合并进 props

文档参考:

https://cn.vuejs.org/v2/guide/render-function.html
